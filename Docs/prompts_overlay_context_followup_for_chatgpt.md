# 장중 실행 결과 통합 갱신 프롬프트

새로 제공된 `decision_bundle_v2.json`과 `strategy_table_ko.md`를 읽고 같은 대화의 이전 답변을 **완전히 대체하는 최신 투자 전략**을 작성한다.

## 핵심 원칙

1. delta만 설명하지 말고 현재 시점의 완결된 종목별 전략표를 다시 제시한다.
2. `quality.decision_ready=false`, `conditional_strategy_ready=true`이면 실제 현재가·VWAP·상대 거래량을 사용한 조건부 전략을 제시하되 즉시 주문으로 승격하지 않는다.
3. 두 전역 준비도 값이 모두 false여도 개별 `execution_ready` 또는 `conditional_strategy_ready` 행이 있으면 혼합 모드로 처리하고 유효한 행을 숨기지 않는다. 두 종류의 유효 행이 모두 0개일 때만 전체 데이터 장애로 처리한다.
4. `generated_in_current_run=false`, `PRIOR_SESSION_BACKFILL`, `HISTORICAL_REFERENCE_ONLY` 행은 현재 주문 근거로 사용하지 않는다.
5. `current_execution_promotion!=POSSIBLE` 또는 provider 상태 재확인이 필요한 행은 즉시 행동으로 승격하지 않는다.
6. 이전 답변과 충돌하면 최신 유효 bundle을 우선하고 변경 이유를 적는다.
6. 보유 종목의 매도·축소 위험을 신규 매수 후보보다 먼저 배치한다.
7. 같은 결측값을 여러 표에서 반복하지 않는다.
8. 모든 투자자용 분류는 한국어로 쓰고 영문 내부 코드는 노출하지 않는다.

## 한국어 전략 분류

- 지금 분할매수 검토
- 조건 확인 후 분할매수 검토
- 보유 유지
- 비중 축소 검토
- 매도·청산 검토
- 종가 확인 후 판단
- 조건 충족 전 대기
- 신규 매수 회피
- 데이터 확인 전 대기

## 출력

### 1. 갱신 결론

최신 기준시각과 가장 중요한 전략 변화 한 가지를 한 문장으로 쓴다.

### 2. 최신 데이터 상태

- 대체하는 이전 run 또는 bundle
- 최신 실행 run
- checkpoint와 시장 데이터 기준시각
- 현재 세션 핵심 데이터 충족률
- 조건부 전략 가능 비율
- 실행 가능 여부

### 3. 최신 종목별 투자 전략표

보유 종목을 모두 포함하고 신규 후보는 상위 5개까지만 표시한다.

| 우선순위 | 종목 | 보유 여부 | 행 모드 | 최신 전략 | 현재가·기준시각 | VWAP 대비 | 상대 거래량·거래대금 | 섹터·지수 동조 | 실행 조건 | 위험·무효화 | 이전 대비 변화 |
|---:|---|---|---|---|---|---|---|---|---|---|---|

### 4. 지금 할 일

즉시 확인할 3~5개 행동만 적는다.

### 5. 변경된 종목

전략이 바뀐 종목과 `왜 바뀌었는지`를 가격, VWAP, 상대 거래량, 지지선 중 실제 제공된 근거로 설명한다.

### 6. 다음 checkpoint

다음 확인 시각과 전략을 바꿀 기준을 적는다.

최신 bundle에 행별 실행 가능·조건부 행이 모두 0개인 데이터 장애 상태라면 3번 전략표 대신 `영향 종목 / 누락 필드 / 마지막 유효 기준시각 / 다음 재시도`의 짧은 장애표를 한 번만 제시한다.

모든 답변은 한국어로 작성한다.
