# 한국투자 HTS 연동 미채택 결정

## 결론

TradingAgents는 eFriend Plus를 포함한 HTS 화면 제어·캡처·OCR 연동을 사용하지 않는다. 계좌와 시장 데이터의 정본은 KIS Open API 및 검증 가능한 구조화 데이터로 한정한다.

2026-07-17과 2026-07-23 로컬 검증에서 eFriend Plus 창을 탐지할 수는 있었지만, Windows 접근성 트리가 세부 컨트롤과 값을 안정적으로 제공하지 않았고 로그인·권한 확인 같은 사용자 전용 단계도 필요했다. 좌표·OCR 자동화는 해상도, 창 배치, 팝업, 버전, 로그인 상태에 따라 깨지기 쉬우며 수치 출처와 재현성을 보장하기 어렵다. 따라서 얻을 수 있는 일부 화면 전용 정보보다 운영 위험이 크다고 판단해 도입하지 않는다.

## 데이터 획득 우선순위

1. **KIS Open API를 계좌·체결·시세의 정본으로 사용한다.** 기존 `tradingagents/portfolio/kis.py`, `tradingagents/dataflows/intraday/kis_quotes.py`, `tradingagents/dataflows/intraday/microstructure.py` 경로를 유지한다.
2. **공식 공시와 구조화된 시장 데이터로 보완한다.** API에 없는 기업 이벤트와 투자 근거는 출처·관측 시각을 기록할 수 있는 데이터만 사용한다.
3. **사용자가 직접 제공한 내보내기 파일은 선택적 보조 입력으로만 사용한다.** CSV/Excel/JSON을 처리할 때도 출처와 기준 시각을 기록하고 KIS API 값과 자동 혼합하지 않는다.
4. **HTS 화면은 데이터 수집 또는 주문 수단으로 사용하지 않는다.** 창 활성화, 메뉴 이동, 화면 캡처, OCR, 키보드·마우스 자동화, 주문 입력을 프로젝트 실행 흐름에 포함하지 않는다.

## 운영 계약

- HTS 실행 여부는 어떤 workflow의 선행 조건도 아니다.
- HTS가 꺼져 있거나 로그인되지 않아도 daily 분석, overlay, Work 종합, Pages 게시가 정상 실행되어야 한다.
- HTS용 비밀번호, 인증서, OTP, 계좌 식별자, 화면 캡처를 저장하거나 수집하지 않는다.
- KIS API 또는 구조화 데이터가 없으면 화면에서 추정하지 않고 `DATA_UNAVAILABLE` 또는 `NEEDS_LIVE_RECHECK`로 표시한다.
- 사용자가 제공한 파일과 API 값이 충돌하면 자동 선택하지 않고 출처별 값을 분리해 표시한다.
- 향후에도 HTS 자동화 도입은 기본 개선 항목이나 미완료 작업으로 간주하지 않는다. 정책 변경은 사용자의 새로운 명시적 지시가 있을 때만 검토한다.

## HTS 대신 추가로 확보할 정보

- 계좌: 잔고, 미체결, 체결, 기간손익, 권리 이벤트는 KIS 계좌 API를 사용한다.
- 실시간 판단: 현재가, 호가, 체결강도·거래량 등 지원되는 미시구조 데이터는 KIS 시세 경로를 사용한다.
- 기업·시장 이벤트: 거래소·공시·기업 IR 등 출처가 명확한 데이터를 사용한다.
- HTS 전용 조건검색식이나 사용자 레이아웃은 자동 수집하지 않는다. 필요하면 동일 조건을 코드로 명시해 재현 가능한 scanner로 구현한다.

이 문서는 기존의 “HTS 읽기 전용 보조 연동” 계획을 대체하는 미채택 결정 기록이다.
